PROJECT GOAL:
Refactor the existing ReadyTMS application so it supports multiple separate companies (multi-tenant architecture) without breaking the current Ready Carrier setup.

BUSINESSES TO SUPPORT:
1. Ready Carrier = existing trucking/dispatch company
2. North East Express = second separate trucking company with its own users, approvals, and branding
3. ReadyTMS = the software platform itself, which should become a standalone SaaS product that can later be sold to outside companies

IMPORTANT RULES:
- Do NOT rebuild the whole app from scratch
- Do NOT break current Ready Carrier functionality
- Do NOT duplicate the whole codebase into multiple apps
- Make changes in safe phases
- Keep backward compatibility where possible
- Before each major change, explain what files/tables will be changed
- After each phase, run tests and summarize what still works and what needs attention

OVERALL ARCHITECTURE TARGET:
- One shared platform/app
- Multiple separate company workspaces
- Each company only sees its own data
- Each company has its own branding, users, approval workflow, and settings
- ReadyTMS becomes the parent platform
- Ready Carrier and North East Express become tenants/accounts inside the platform

PHASE 0 — ANALYZE CURRENT APP FIRST
Step 0.1:
Scan the existing codebase and identify:
- current framework and stack
- authentication method
- database schema
- all places where “Ready Carrier” is hardcoded
- all major business entities (users, drivers, loads, approvals, invoices, documents, settings, etc.)
- all routes/pages/components that assume only one company exists

Step 0.2:
Create a short audit report before changing code:
- list of files with hardcoded branding
- list of database tables that need company separation
- list of risky areas
- migration strategy proposal

Do not start refactoring until this audit is shown.
